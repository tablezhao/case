## 🚀 使用指南

### 快速开始

1. **准备Excel文件**
   - 第一行：列名（通报日期、应用名称等）
   - 第二行开始：数据

2. **必需字段**
   - ⭐ 通报日期（必需）
   - ⭐ 应用名称（必需）

3. **可选字段**
   - 开发者/运营者
   - 监管部门
   - 应用平台
   - 主要违规内容
   - 原文链接

4. **执行导入**
   - 进入"案例管理"页面
   - 点击"导入"按钮
   - 选择Excel文件
   - 等待导入完成

### 日期格式建议

✅ **推荐使用**：
- `2024-01-15`（最标准）
- `2024/01/15`
- `2024年1月15日`

❌ **避免使用**：
- 不完整的日期（如"1月15日"）
- 文本描述（如"上个月"）

## ⚠️ 常见问题

### Q1：日期格式错误怎么办？

**A**：系统现在支持多种日期格式，包括：
- 标准格式：`2024-01-15`
- 斜杠格式：`2024/01/15`
- 中文格式：`2024年1月15日`
- Excel序列号：自动识别

如果仍然报错，请确保日期字段不为空。

### Q2：部分数据导入失败怎么办？

**A**：
1. 查看错误信息，找到具体的行号
2. 在Excel中定位到该行
3. 修正错误数据
4. 重新导入

系统会自动去重，不用担心重复导入。

### Q3：如何知道导入了哪些数据？

**A**：导入完成后，系统会显示详细报告：
- 成功导入的数量
- 去重的数量
- 新增的部门和平台
- 跳过的数据（如果有）

### Q4：可以导入多少条数据？

**A**：
- 建议每次不超过1000条
- 大量数据建议分批导入
- 系统会自动处理去重
