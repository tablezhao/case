# App监管案例查询平台升级需求文档

## 1. 网站概述

### 1.1 网站名称
合规通\n
### 1.2 网站描述
构建一个信息透明、查询便捷的平台,用于展示各级网络监管部门发布的App违规监管案例和监管资讯。通过数据可视化清晰呈现监管动态、地域分布及违规类型趋势,同时提供强大的后台数据管理能力和智能案例导入功能,确保信息的准确性和时效性。

## 2. 功能需求

### 2.1 前端用户界面功能

#### 2.1.1 首页核心模块
\n##### 2.1.1.1 统计概览（核心数据总览面板）
\n- 本月通报频次及涉及应用数量
- 本季度通报频次及涉及应用数量
- 本年度通报频次及涉及应用数量
- 累计通报案例总数展示
- 累计涉及的应用总数统计
- 最近一次通报的发布日期显示
- 最近一次通报的监管部门名称展示

注：所有时间维度(本月、本季度、本年度)下的通报频次和涉及应用统计逻辑、计算口径必须严格保持一致\n
##### 2.1.1.2 监管趋势分析（年度月度趋势图）

- 年度及月度通报案例数量趋势折线图
- 支持按年度视图和月度视图切换查看
- 展示通报频次的时间变化规律
\n##### 2.1.1.3 应用平台分布\n
- 被通报应用的来源平台分布图
- 使用饼图或柱状图展示各平台占比情况
- 支持点击查看对应平台的详细案例

##### 2.1.1.4 近期监管资讯

- 展示最新发布的监管资讯列表
- 包含资讯标题、发布日期、发布部门\n- 提供〖查看详情〗链接跳转到完整资讯内容

#### 2.1.2 数据可视化分析模块

- 通报部门所在地理分布地图(中国地图轮廓)\n- 各通报部门出现频次分布图(饼图/柱状图)
- 热点违规问题词云\n\n#### 2.1.3 案例详情查询系统

- **关键词检索功能**：\n  - 在案例查询页面顶部显著位置设置搜索框
  - 支持输入关键词检索案例标题、摘要、编号等核心文本信息
  - 用户可通过点击〖搜索〗按钮或按下回车键触发检索
  - 提供清空输入框的快捷按钮
  - 检索后页面刷新,仅展示包含关键词的案例列表
  - 无匹配结果时显示〖未找到相关案例〗提示

-可排序、可分页的案例列表展示
- 案例条目包含信息：通报发布日期、被通报应用名称、应用开发者/运营者、发出通报的监管部门全称、应用来源平台、主要违规问题摘要
- 操作按钮：〖查看详情〗(模态框或新页面详细违规内容)、〖原文链接〗(新窗口打开官方原始通报网页)

#### 2.1.4 监管资讯查询系统

-可排序、可分页的监管资讯列表展示\n- 案例条目包含信息：监管资讯发布日期、发出监管资讯的监管部门、监管资讯摘要\n- 操作按钮：〖查看详情〗(模态框或新页面详细监管资讯内容)、〖原文链接〗(新窗口打开官方原始通报网页)

#### 2.1.5 违规问题分析模块

- **功能定位**：
  - 集成在趋势分析页面内,作为独立的〖违规问题分析〗面板或标签页
  - 从宏观趋势深入到微观归因,帮助运营人员快速定位问题根源
\n- **多维度分析指标**：
  - **违规类型分析**：统计各类违规(如内容违规、行为违规、交易违规等)的数量、占比及变化趋势
  - **用户群体分析**：按用户属性(如新/老用户、会员等级、注册渠道等)分析违规分布情况
  - **时间趋势分析**：按小时、日、周等不同粒度,分析违规发生的时间规律
  - **内容/场景分析**：分析违规高发的具体内容板块、功能场景或商品类别
  - **严重程度分析**：按违规的严重等级(如轻度、中度、严重)进行划分和统计

- **数据可视化与交互**：
  - 使用组合图表呈现分析结果：
    +饼图展示违规类型分布
    + 柱状图展示时间趋势
    + 表格展示明细数据
  - 支持数据下钻功能：点击某类违规的统计数字,可查看对应的违规记录列表
  - 提供数据筛选功能：支持按维度自由组合筛选(如选择特定时间范围、违规类型、用户群体等)
  - 支持导出分析报表(Excel/CSV格式)

- **非功能性要求**：
  - 数据展示接近实时,延迟不超过设定的数据更新周期
  - 界面设计与现有趋势分析页面保持风格一致
\n#### 2.1.6 模块可见性响应规则

- 前端根据后台模块管理配置动态控制各功能模块的显示状态
- 当某模块被后台关闭时,其对应的导航入口、页面链接或内容区域对用户完全隐藏
- 前端布局自适应调整,隐藏模块后页面自动优化排版,避免出现空白区域
- 支持实时响应后台配置变更,无需刷新页面即可生效

### 2.2 后台管理系统功能
\n#### 2.2.1 智能案例导入中心

- **多模态输入支持**：
  - **文本输入**：用户可直接粘贴或输入一段文字描述,系统自动识别并提取关键信息
  - **图片上传**：支持JPG、PNG等图片格式,AI模型识别图片中的文字信息并提取
  - **PDF文档上传**：支持PDF文件,自动解析文档内容提取结构化数据
\n- **AI信息处理流程**：
  - 集成多模态理解大模型插件,自动识别输入内容中的关键信息
  - 提取当事人姓名、时间、地点、事件描述等预设字段信息
  - 自动填充到案例编辑界面的对应位置
  - 提供清晰的预览界面供用户核对和手动修改

- **批量处理能力**：
  - 支持同时处理多个输入源\n  - 批量提取和填充案例信息
  - 批量导入到数据库\n
#### 2.2.2 案例数据管理中心

- 案例数据完整CRUD操作界面
- 支持上传Excel/CSV文件批量导入数据
- 支持筛选或全部数据导出为常用格式文件
- **监管部门与应用平台快速新增功能**：
  - 在案例编辑界面中,〖监管部门〗和〖应用平台〗字段采用可输入的下拉选择框\n  - 当用户输入的部门或平台名称在系统中不存在时,下拉列表自动显示〖新增：[用户输入名称]〗选项
  - 用户选择新增选项后,系统自动将该名称作为新记录保存至〖部门与平台〗管理模块的对应分类中
  - 新增操作无需离开当前编辑页面,保存案例时同步完成新记录的创建
  - 操作完成后显示成功提示,告知用户新记录已保存至主数据模块
  - 新增的部门或平台记录初始仅包含名称字段,后续可在〖部门与平台〗管理模块中补充完善详细信息

- **案例筛选与检索功能**：
  - 在案例管理页面顶部设置筛选区域,提供以下筛选条件：
    + 通报日期范围选择(起始日期-结束日期)
    + 监管部门下拉选择框(支持多选或单选)
    + 应用平台下拉选择框(支持多选或单选)
    + 应用名称/开发者文本输入框\n  - 提供关键词检索功能：
    + 在筛选区域设置搜索框,支持输入关键词检索案例标题、摘要、编号、违规内容等字段
    + 用户可通过点击〖搜索〗按钮或按下回车键触发检索
    + 提供〖重置〗按钮清空所有筛选条件和关键词
  - 筛选和检索可组合使用,结果实时更新案例列表
  - 筛选或检索后的列表仍支持排序和分页功能
  - 无匹配结果时显示〖未找到符合条件的案例〗提示

#### 2.2.3 监管资讯管理中心

- 案例数据完整CRUD操作界面
- 支持上传Excel/CSV文件批量导入数据
- 支持筛选或全部数据导出为常用格式文件
\n#### 2.2.4 前台模块可见性管理中心

- **功能定位**：
  - 提供集中的模块管理界面,用于控制前台各功能模块的显示/隐藏状态
  - 管理员可独立控制每个模块的可见性,实现灵活的页面布局管理
\n- **可控模块列表**：
  - 统计概览（核心数据总览面板）
  - 监管趋势分析（年度月度趋势图）\n  - 应用平台分布\n  - 近期监管资讯
  - 案例查询模块
  - 监管资讯模块
  - 监管部门模块
  - 趋势分析模块
  - 问题分析模块
\n- **管理界面设计**：
  - 采用列表式布局,每个模块独立成行\n  - 每行显示模块名称和状态开关(Toggle按钮或复选框)
  - 开关状态清晰标识：开启(蓝色/绿色)、关闭(灰色)\n  - 支持批量操作：全部开启、全部关闭功能按钮
\n- **操作规则**：
  - 点击开关按钮即时切换模块状态
  - 状态变更后自动保存至数据库,无需额外确认
  - 操作后显示简洁的成功提示(〖模块状态已更新〗)
  - 前台同步响应配置变更,延迟不超过设定的缓存刷新周期

- **扩展性设计**：
  - 模块列表采用配置化设计,便于后续新增可控模块
  - 预留模块分组功能入口,支持按功能类型分组管理
\n#### 2.2.5 前台展示配置中心

-拖拽或开关式界面动态配置前台首页各模块显示/隐藏状态及排列顺序
- 支持调整各模块的尺寸比例,确保统计概览、监管趋势分析、应用平台分布、近期监管资讯、违规问题分析、案例详情查询系统在首页中的视觉权重协调平衡
\n#### 2.2.6 静态内容管理

- 富文本编辑器维护更新前台页面各类说明文字、公告等静态信息
\n#### 2.2.7 部门与平台管理模块

- 提供独立的管理界面,用于维护监管部门和应用平台的主数据\n- 支持对通过案例编辑界面自动创建的记录进行信息补充和修改
- 包含名称、简称、所属地区、联系方式等字段的完整维护功能

## 3. 技术实现要求

### 3.1 数据准确性与来源

-后端建立数据校验机制确保录入数据规范性
- 所有数据标明权威来源,〖原文链接〗直接指向官方页面
\n### 3.2 AI智能处理模块

- **多模态大模型集成**：
  - 文本处理插件：识别和提取文本中的结构化信息
  - 图片识别插件：OCR技术识别图片文字并提取关键数据
  - PDF解析插件：自动解析PDF文档内容\n\n- **信息映射算法**：
  - 将提取的原始信息自动映射到预设的案例字段中
  - 确保提取的当事人、时间、地点等信息准确填充到对应字段

- **用户反馈机制**：
  - 提供提取结果的可视化预览界面
  - 支持用户对AI提取结果进行手动修正
  - 修正后的数据优先级高于AI提取结果

### 3.3 前端技术选型

- 采用主流前端框架(Vue.js/React)搭配组件库(Element-UI/Ant Design)\n- 数据可视化使用ECharts或AntV G2图表库
- 采用单页面应用(SPA)架构\n- 实现模块动态加载机制,根据后台配置控制组件的渲染与隐藏

### 3.4 后端数据处理逻辑

- 建议技术栈：Python(Django/Flask)、Java(Spring Boot)或Node.js\n- 设计RESTful API接口：
  - 获取核心统计数据接口(包含本月、本季度、本年度维度)
  - 获取可视化图表数据接口(按年、月、地域、部门等维度)
  - 获取首页统计概览数据接口
  - 获取监管趋势分析数据接口(年度和月度)
  - 获取应用平台分布数据接口
  - 获取近期监管资讯列表接口
  - 分页查询案例列表接口\n  - 关键词检索案例接口(支持标题、摘要、编号等字段模糊匹配)
  - 获取单个案例详情接口\n  - AI智能解析接口(接收多种格式输入,返回结构化数据结果)
  - 监管部门与应用平台的新增、查询、更新接口
  - 案例筛选与复合查询接口(支持日期范围、部门、平台、关键词等多条件组合查询)
  - 违规问题分析数据接口(支持按违规类型、用户群体、时间粒度、场景、严重程度等维度统计)
  - 违规记录下钻查询接口(根据分析维度返回对应的详细记录列表)
  - **模块可见性配置接口**：
    + 获取所有模块的当前状态接口(返回模块名称和开启/关闭状态)
    + 更新单个模块状态接口(接收模块ID和目标状态)
    + 批量更新模块状态接口(接收模块ID列表和目标状态)\n- 实现Redis数据缓存策略提高高频数据访问速度
- 模块配置采用独立的缓存键,配置变更后立即清除对应缓存
\n### 3.5 数据库设计

- 核心数据表：\n  - `cases`(案例主表)：存储案例基本信息和违规摘要
  - `case_details`(案例详情表)：与主表关联,存储详细违规描述\n  - `regulatory_departments`(监管部门表)：存储部门信息实现标准化,包含名称、简称、所属地区等字段
  - `app_platforms`(应用平台表)：存储平台类型实现标准化,包含名称、平台类型等字段
  - `module_config`(模块配置表)：存储前台模块的可见性状态,包含模块ID、模块名称、是否启用、更新时间等字段
- 建立合适索引优化查询性能,特别针对标题、摘要、编号、通报日期、监管部门、应用平台、违规类型等字段建立索引支持快速检索和筛选

### 3.6 性能与稳定性

- 前端代码打包优化减少初始加载时间
- 后端API性能测试与优化确保高并发响应速度
- 考虑使用CDN加速静态资源访问
- AI模型调用的异步处理和超时机制
- 违规问题分析模块的数据查询优化,确保复杂统计查询的响应时间在可接受范围内
- 模块配置接口采用缓存机制,配置读取性能优化至毫秒级

## 4. 设计风格\n
### 4.1 配色方案

主色调采用政府机构蓝(#1E4A8C)搭配白色背景,辅助色使用警示橙色(#FF6B35)突出违规信息

### 4.2 视觉细节

采用4px圆角设计,使用卡片式布局增强层次感,图标风格统一采用线性图标\n
### 4.3 整体布局

首页采用仪表盘式布局,各功能模块(统计概览、监管趋势分析、应用平台分布、近期监管资讯、违规问题分析、案例查询)通过灵活的网格系统排布,确保视觉比例协调,支持后台配置模块尺寸和位置\n
