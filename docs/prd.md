# App监管案例查询平台需求文档

## 1. 网站概述

### 1.1 网站名称

合规通

### 1.2 网站描述

构建一个信息透明、查询便捷的平台，用于展示各级网络监管部门发布的App违规监管案例和监管资讯。通过数据可视化清晰呈现监管动态、地域分布及违规类型趋势，同时提供强大的后台数据管理能力，确保信息的准确性和时效性。

## 2. 功能需求

### 2.1 前端用户界面功能

#### 2.1.1 核心数据总览面板

- 累计通报案例总数展示
- 累计涉及的应用总数统计
- 最近一次通报的发布日期显示
- 最近一次通报的监管部门名称展示
- 近期监管资讯

#### 2.1.2 数据可视化分析模块

- 年度及月度通报案例数量趋势折线图
- 通报部门所在地理分布地图（中国地图轮廓）
- 各通报部门出现频次分布图（饼图/柱状图）
- 被通报应用的来源平台分布图
- 热点违规问题词云

#### 2.1.3 案例详情查询系统

- 可排序、可分页的案例列表展示
- 案例条目包含信息：通报发布日期、被通报应用名称、应用开发者/运营者、发出通报的监管部门全称、应用来源平台、主要违规问题摘要
- 操作按钮：\"查看详情\"（模态框或新页面详细违规内容）、「原文链接」（新窗口打开官方原始通报网页）

#### 2.1.4 监管资讯查询系统

- 可排序、可分页的监管资讯列表展示
- 案例条目包含信息：监管资讯发布日期、发出监管资讯的监管部门、监管资讯摘要
- 操作按钮：\"查看详情\"（模态框或新页面详细监管资讯内容）、「原文链接」（新窗口打开官方原始通报网页）

### 2.2 后台管理系统功能

#### 2.2.1 案例数据管理中心

- 案例数据完整CRUD操作界面
- 支持上传Excel/CSV文件批量导入数据
- 支持筛选或全部数据导出为常用格式文件

#### 2.2.2 监管资讯管理中心

- 案例数据完整CRUD操作界面
- 支持上传Excel/CSV文件批量导入数据
- 支持筛选或全部数据导出为常用格式文件

#### 2.2.2 前端展示配置中心

- 拖拽或开关式界面动态配置前端首页各模块显示/隐藏状态及排列顺序

#### 2.2.3 静态内容管理

- 富文本编辑器维护更新前端页面各类说明文字、公告等静态信息

## 3. 技术实现要求

### 3.1 数据准确性与来源

- 后端建立数据校验机制确保录入数据规范性
- 所有数据标明权威来源，「原文链接」直接指向官方页面

### 3.2 前端技术选型

- 采用主流前端框架（Vue.js/React）搭配组件库（Element-UI/Ant Design）
- 数据可视化使用ECharts或AntV G2图表库
- 采用单页面应用（SPA）架构

### 3.3 后端数据处理逻辑

- 建议技术栈：Python（Django/Flask）、Java（Spring Boot）或Node.js
- 设计RESTful API接口：
  - 获取核心统计数据接口
  - 获取可视化图表数据接口（按年、月、地域、部门等维度）
  - 分页查询案例列表接口
  - 获取单个案例详情接口
- 实现Redis数据缓存策略提高高频数据访问速度

### 3.4 数据库设计

- 核心数据表：
  - `cases`（案例主表）：存储案例基本信息和违规摘要
  - `case_details`（案例详情表）：与主表关联，存储详细违规描述
  - `regulatory_departments`（监管部门表）：存储部门信息实现标准化
  - `app_platforms`（应用平台表）：存储平台类型实现标准化
- 建立合适索引优化查询性能

### 3.5 性能与稳定性

- 前端代码打包优化减少初始加载时间
- 后端API性能测试与优化确保高并发响应速度
- 考虑使用CDN加速静态资源访问

## 4. 设计风格

### 4.1 配色方案

主色调采用政府机构蓝（#1E4A8C）搭配白色背景，辅助色使用警示橙色（#FF6B35）突出违规信息

### 4.2 视觉细节

采用4px圆角设计，使用卡片式布局增强层次感，图标风格统一采用线性图标

### 4.3 整体布局

首页采用仪表盘式布局，左侧数据总览面板，右侧数据可视化区域，底部案例列表区域
