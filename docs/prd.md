# App监管案例查询平台升级需求文档

## 1. 网站概述

### 1.1 网站名称
\n合规通\n
### 1.2 网站描述

构建一个信息透明、查询便捷的平台,用于展示各级网络监管部门发布的App违规监管案例和监管资讯。通过数据可视化清晰呈现监管动态、地域分布及违规类型趋势,同时提供强大的后台数据管理能力和智能案例导入功能,确保信息的准确性和时效性。

## 2. 功能需求

### 2.1 前端用户界面功能

#### 2.1.1 核心数据总览面板

-本月通报频次及涉及应用数量
- 本季度通报频次及涉及应用数量
- 本年度通报频次及涉及应用数量
- 累计通报案例总数展示
- 累计涉及的应用总数统计
- 最近一次通报的发布日期显示
- 最近一次通报的监管部门名称展示
- 近期监管资讯\n
注：所有时间维度（本月、本季度、本年度）下的通报频次和涉及应用统计逻辑、计算口径必须严格保持一致\n
#### 2.1.2 数据可视化分析模块

- 年度及月度通报案例数量趋势折线图
- 通报部门所在地理分布地图（中国地图轮廓）
- 各通报部门出现频次分布图（饼图/柱状图）
- 被通报应用的来源平台分布图
- 热点违规问题词云\n\n#### 2.1.3 案例详情查询系统

- 可排序、可分页的案例列表展示
- 案例条目包含信息：通报发布日期、被通报应用名称、应用开发者/运营者、发出通报的监管部门全称、应用来源平台、主要违规问题摘要\n- 操作按钮：〖查看详情〗（模态框或新页面详细违规内容）、〖原文链接〗（新窗口打开官方原始通报网页）\n\n#### 2.1.4 监管资讯查询系统

- 可排序、可分页的监管资讯列表展示
- 案例条目包含信息：监管资讯发布日期、发出监管资讯的监管部门、监管资讯摘要\n- 操作按钮：〖查看详情〗（模态框或新页面详细监管资讯内容）、〖原文链接〗（新窗口打开官方原始通报网页）

### 2.2 后台管理系统功能

#### 2.2.1 智能案例导入中心

- **多模态输入支持**：\n  - **文本输入**：用户可直接粘贴或输入一段文字描述,系统自动识别并提取关键信息
  - **图片上传**：支持JPG、PNG等图片格式,AI模型识别图片中的文字信息并提取
  - **PDF文档上传**：支持PDF文件,自动解析文档内容提取结构化数据
\n- **AI信息处理流程**：
  - 集成多模态理解大模型插件,自动识别输入内容中的关键信息
  - 提取当事人姓名、时间、地点、事件描述等预设字段信息
  - 自动填充到案例编辑界面的对应位置
  - 提供清晰的预览界面供用户核对和手动修改

- **批量处理能力**：
  - 支持同时处理多个输入源\n  - 批量提取和填充案例信息
  - 批量导入到数据库\n
#### 2.2.2 案例数据管理中心

- 案例数据完整CRUD操作界面
- 支持上传Excel/CSV文件批量导入数据
- 支持筛选或全部数据导出为常用格式文件
- **监管部门与应用平台快速新增功能**：
  - 在案例编辑界面中,〖监管部门〗和〖应用平台〗字段采用可输入的下拉选择框\n  - 当用户输入的部门或平台名称在系统中不存在时,下拉列表自动显示〖新增：[用户输入名称]〗选项
  - 用户选择新增选项后,系统自动将该名称作为新记录保存至〖部门与平台〗管理模块的对应分类中
  - 新增操作无需离开当前编辑页面,保存案例时同步完成新记录的创建
  - 操作完成后显示成功提示,告知用户新记录已保存至主数据模块
  - 新增的部门或平台记录初始仅包含名称字段,后续可在〖部门与平台〗管理模块中补充完善详细信息

#### 2.2.3 监管资讯管理中心

- 案例数据完整CRUD操作界面
- 支持上传Excel/CSV文件批量导入数据
- 支持筛选或全部数据导出为常用格式文件
\n#### 2.2.4 前台展示配置中心

-拖拽或开关式界面动态配置前台首页各模块显示/隐藏状态及排列顺序
- 支持调整各模块的尺寸比例,确保核心数据总览面板、数据可视化分析模块、案例详情查询系统、监管资讯查询系统在首页中的视觉权重协调平衡

#### 2.2.5 静态内容管理

- 富文本编辑器维护更新前台页面各类说明文字、公告等静态信息
\n#### 2.2.6 部门与平台管理模块

- 提供独立的管理界面,用于维护监管部门和应用平台的主数据
- 支持对通过案例编辑界面自动创建的记录进行信息补充和修改
- 包含名称、简称、所属地区、联系方式等字段的完整维护功能

## 3. 技术实现要求

### 3.1 数据准确性与来源\n
-后端建立数据校验机制确保录入数据规范性
- 所有数据标明权威来源,〖原文链接〗直接指向官方页面
\n### 3.2 AI智能处理模块

- **多模态大模型集成**：
  - 文本处理插件：识别和提取文本中的结构化信息
  - 图片识别插件：OCR技术识别图片文字并提取关键数据
  - PDF解析插件：自动解析PDF文档内容
\n- **信息映射算法**：
  - 将提取的原始信息自动映射到预设的案例字段中
  - 确保提取的当事人、时间、地点等信息准确填充到对应字段

- **用户反馈机制**：
  - 提供提取结果的可视化预览界面
  - 支持用户对AI提取结果进行手动修正
  - 修正后的数据优先级高于AI提取结果

### 3.3 前端技术选型

- 采用主流前端框架（Vue.js/React）搭配组件库（Element-UI/Ant Design）
- 数据可视化使用ECharts或AntV G2图表库
- 采用单页面应用（SPA）架构
\n### 3.4 后端数据处理逻辑

- 建议技术栈：Python（Django/Flask）、Java（Spring Boot）或Node.js
- 设计RESTful API接口：
  - 获取核心统计数据接口（包含本月、本季度、本年度维度）
  - 获取可视化图表数据接口（按年、月、地域、部门等维度）
  - 分页查询案例列表接口
  - 获取单个案例详情接口\n  - AI智能解析接口（接收多种格式输入,返回结构化数据结果）
  - 监管部门与应用平台的新增、查询、更新接口
- 实现Redis数据缓存策略提高高频数据访问速度
\n### 3.5 数据库设计

- 核心数据表：\n  - `cases`（案例主表）：存储案例基本信息和违规摘要
  - `case_details`（案例详情表）：与主表关联,存储详细违规描述
  - `regulatory_departments`（监管部门表）：存储部门信息实现标准化,包含名称、简称、所属地区等字段
  - `app_platforms`（应用平台表）：存储平台类型实现标准化,包含名称、平台类型等字段
- 建立合适索引优化查询性能
\n### 3.6 性能与稳定性

- 前端代码打包优化减少初始加载时间
- 后端API性能测试与优化确保高并发响应速度
- 考虑使用CDN加速静态资源访问
- AI模型调用的异步处理和超时机制

## 4. 设计风格\n
### 4.1 配色方案

主色调采用政府机构蓝（#1E4A8C）搭配白色背景,辅助色使用警示橙色（#FF6B35）突出违规信息\n
### 4.2 视觉细节

采用4px圆角设计,使用卡片式布局增强层次感,图标风格统一采用线性图标\n
### 4.3 整体布局

首页采用仪表盘式布局,各功能模块（核心数据总览面板、数据可视化分析、案例查询、资讯查询）通过灵活的网格系统排布,确保视觉比例协调,支持后台配置模块尺寸和位置\n
