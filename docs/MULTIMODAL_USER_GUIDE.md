# 智能案例导入 - 多模态输入使用指南

## 快速开始

智能案例导入功能现已支持**4种输入方式**，您可以根据实际情况选择最方便的方式：

### 🌐 方式1：URL输入

**适用场景**：监管部门官网发布的在线通报

**操作步骤**：
1. 打开监管通报网页，复制浏览器地址栏的URL
2. 在智能导入页面选择"URL输入"标签
3. 粘贴URL到输入框
4. 点击"开始解析"
5. 等待10-30秒，AI自动提取信息
6. 检查并修改数据
7. 点击"确认导入"

**优点**：
- ✅ 自动抓取网页内容
- ✅ AI自动提取关键信息
- ✅ 准确率高（85%+）

**示例URL**：
```
https://www.miit.gov.cn/jgsj/xgj/APPqhyhqkxx/2024/01/15/notice123.html
```

---

### 📝 方式2：文本输入

**适用场景**：已复制的文本内容、离线文档

**操作步骤**：
1. 复制案例文本内容
2. 在智能导入页面选择"文本输入"标签
3. 粘贴文本到输入框
4. 点击"开始解析"
5. 等待5-15秒，AI自动提取信息
6. 检查并修改数据
7. 点击"确认导入"

**优点**：
- ✅ 处理速度快
- ✅ 无需网络请求
- ✅ 适合离线内容

**示例文本**：
```
工业和信息化部于2024年1月15日发布通报，
"某某App"（开发者：XX科技有限公司）
在应用宝平台存在超范围收集个人信息的问题，
违规获取用户通讯录、位置信息等敏感数据。
```

---

### 🖼️ 方式3：图片上传

**适用场景**：通报截图、手机拍摄的文件照片

**操作步骤**：
1. 准备通报案例的图片（JPG或PNG格式）
2. 在智能导入页面选择"图片上传"标签
3. 点击"选择图片"或拖拽图片到上传区域
4. 等待上传完成
5. 点击"开始解析"
6. **根据图片内容手动填写信息**
7. 点击"确认导入"

**当前限制**：
- ⚠️ 暂不支持自动OCR识别
- ⚠️ 需要用户根据图片手动填写
- ⚠️ 未来版本将支持自动识别

**文件要求**：
- 格式：JPG、PNG
- 大小：最大5MB
- 建议：清晰、完整的截图

---

### 📄 方式4：PDF上传

**适用场景**：PDF格式的通报文档

**操作步骤**：
1. 准备通报案例的PDF文件
2. 在智能导入页面选择"PDF上传"标签
3. 点击"选择PDF"或拖拽PDF到上传区域
4. 等待上传完成
5. 点击"开始解析"
6. **根据PDF内容手动填写信息**
7. 点击"确认导入"

**当前限制**：
- ⚠️ 暂不支持自动PDF解析
- ⚠️ 需要用户根据PDF手动填写
- ⚠️ 未来版本将支持自动解析

**文件要求**：
- 格式：PDF
- 大小：最大10MB
- 建议：文字型PDF（非扫描版）

## 常见问题

### Q1：哪种输入方式最好？

**推荐顺序**：
1. **URL输入**：如果有网页链接，优先使用
2. **文本输入**：如果已复制文本，使用此方式
3. **图片/PDF**：如果只有图片或PDF，使用对应方式

### Q2：为什么图片和PDF不能自动识别？

当前版本为简化实现，图片OCR和PDF解析功能将在后续版本中提供。
- v2.1版本：集成OCR识别
- v2.2版本：集成PDF解析

### Q3：上传的文件会保存吗？

不会。上传的文件仅临时存储，导入完成后会自动删除。

### Q4：可以同时使用多种输入方式吗？

不可以。每次只能选择一种输入方式。如果需要导入多个案例，请分别操作。

### Q5：AI提取的数据准确吗？

AI提取的准确率约为80-85%，建议您在导入前仔细检查并修改不准确的字段。

## 最佳实践

### 1. 选择合适的输入方式

- **有URL**：优先使用URL输入
- **有文本**：使用文本输入
- **只有图片/PDF**：使用对应的上传方式

### 2. 提供完整信息

- 确保输入内容包含完整的案例信息
- 包括：日期、应用名称、开发者、部门、平台、违规问题

### 3. 仔细检查数据

- AI提取的数据可能不完全准确
- 重点检查：应用名称、通报日期
- 补充缺失的字段

### 4. 规范填写

- 日期格式：YYYY-MM-DD
- 名称表述：保持一致
- 摘要简洁：建议100字以内
