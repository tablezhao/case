# 关键词检索功能快速参考

## 🚀 快速开始

### 用户使用指南

#### 基本搜索
```
1. 打开案例查询页面
2. 在搜索框输入关键词（如"微信"）
3. 点击"搜索"按钮或按回车键
4. 查看搜索结果
```

#### 高级搜索
```
1. 输入关键词
2. 选择日期范围
3. 选择监管部门
4. 选择应用平台
5. 点击搜索
```

#### 使用搜索建议
```
1. 当搜索无结果时
2. 查看蓝色背景的"搜索建议"区域
3. 点击建议词快速搜索
```

## 💡 搜索技巧

### 1. 关键词选择

✅ **推荐做法**：
- 使用核心关键词（如"隐私"而不是"隐私保护措施"）
- 使用常见词汇（如"微信"而不是"微信小程序"）
- 尝试不同的同义词（如"应用"、"APP"、"软件"）

❌ **避免做法**：
- 使用过长的关键词
- 使用过于具体的词汇
- 使用特殊字符

### 2. 组合筛选

✅ **推荐做法**：
- 先用关键词缩小范围
- 再用日期、部门等进一步筛选
- 逐步细化搜索条件

❌ **避免做法**：
- 一次设置过多筛选条件
- 筛选条件过于严格

### 3. 无结果处理

✅ **推荐做法**：
- 查看搜索建议
- 尝试更通用的关键词
- 减少筛选条件
- 检查拼写是否正确

❌ **避免做法**：
- 立即放弃
- 重复相同的搜索

## 📊 功能特性

### ✅ 支持的功能

| 功能 | 说明 |
|------|------|
| 全局搜索 | 搜索所有数据库记录，不限于当前页 |
| 模糊匹配 | 支持部分匹配，如"微"可以匹配"微信" |
| 中文分词 | 支持中文词组拆分匹配 |
| 多字段搜索 | 同时搜索应用名称、开发者、违规内容等 |
| 组合筛选 | 支持关键词+日期+部门+平台组合筛选 |
| 搜索建议 | 无结果时提供相关搜索建议 |
| 相关性排序 | 结果按相关性和日期排序 |
| 分页浏览 | 支持翻页查看所有结果 |

| 功能 | 计划 |

| 搜索历史 | 中期实现 |

## 🎯 常见问题

### Q1：为什么搜索不到结果？

**A**：可能的原因：
1. 关键词拼写错误
2. 使用了过于具体的词汇
3. 筛选条件过于严格
4. 数据库中确实没有相关数据

**解决方法**：
- 检查拼写
- 尝试更通用的关键词
- 减少筛选条件
- 查看搜索建议

### Q2：搜索结果太多怎么办？

**A**：可以通过以下方式缩小范围：
1. 使用更具体的关键词
2. 添加日期范围筛选
3. 选择特定的监管部门
4. 选择特定的应用平台

### Q3：搜索结果不准确怎么办？

**A**：可以尝试：
1. 使用更精确的关键词
2. 使用引号搜索精确短语（待实现）
3. 使用多个关键词组合
4. 查看搜索建议

### Q4：如何搜索多个关键词？

**A**：当前版本暂不支持多关键词搜索，建议：
1. 使用最重要的关键词搜索
2. 然后使用筛选条件进一步过滤
3. 或者分别搜索不同关键词

### Q5：搜索速度慢怎么办？

**A**：可能的原因和解决方法：
1. 网络连接慢 → 检查网络
2. 数据量过大 → 添加筛选条件
3. 服务器负载高 → 稍后再试

## 🔧 技术说明

### 搜索原理

```
用户输入 → 关键词预处理 → 数据库全文搜索 → 相关性排序 → 返回结果
```

### 匹配规则

1. **全文搜索**：使用PostgreSQL全文搜索，支持中文分词
2. **模糊匹配**：使用ILIKE进行模糊匹配，确保不漏掉结果
3. **权重排序**：应用名称 > 开发者 > 违规内容

### 性能指标

- 搜索响应时间：< 500ms
- 支持数据量：> 10000条
- 并发支持：> 100 QPS
