# 多维度分析功能快速参考

## 🚀 快速访问

**页面路径：** `/trend-analysis`

**导航位置：** 顶部导航栏 → 趋势分析

## 📊 四大分析模块

### 1. 趋势分析 📈
- **功能：** 展示部门月度通报频次变化趋势
- **图表：** 折线图 + 数据表格
- **用途：** 识别季节性规律、发现趋势变化

### 2. 排名统计 🏆
- **功能：** 年度和月度部门通报排名 TOP 10
- **展示：** 排名卡片（金银铜标识）
- **用途：** 识别最活跃部门、对比排名差异

### 3. 部门对比 🔄
- **功能：** 多部门（最多5个）月度对比
- **图表：** 柱状图
- **用途：** 对比活跃度、发现协同关系

### 4. 高频时段 🔥
- **功能：** 识别通报活动频繁时段（≥5次）
- **展示：** 时段卡片 + 主要部门
- **用途：** 识别监管高峰期、预测未来趋势

## 🎯 核心参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 年份 | 分析的目标年份 | 当前年份 |
| 月份 | 用于月度排名 | 01月 |
| 部门 | 选择1-5个部门 | 前3个部门 |

## 💡 快速上手

### 场景1：查看某部门全年趋势
```
1. 选择年份：2025
2. 选择部门：工业和信息化部
3. 点击"趋势分析"
4. 查看折线图和数据表
```

### 场景2：对比多个部门
```
1. 选择年份：2025
2. 选择部门：工信部、网信办、病毒中心
3. 点击"部门对比"
4. 查看柱状图对比
```

### 场景3：查看年度排名
```
1. 选择年份：2025
2. 点击"排名统计"
3. 查看年度 TOP 10
4. 对比月度 TOP 10
```

### 场景4：识别高频时段
```
1. 点击"高频时段"
2. 查看通报频繁的月份
3. 查看主要监管部门
4. 分析高峰期原因
```

## 📈 数据说明

### 通报频次统计规则
- **去重方式：** 按"部门+日期"去重
- **统计逻辑：** 同一部门同一天只算1次
- **数据来源：** 案例数据库实时数据

### 图表颜色
- 🔵 主色调：Primary
- 🟢 次色调：Secondary
- 🟣 其他：紫色、橙色等

### 排名标识
- 🥇 第1名：金色
- 🥈 第2名：银色
- 🥉 第3名：铜色
- 📊 其他：灰色

## 🔧 技术信息

### 新增API函数（5个）

1. **getDepartmentTimeTrend** - 部门时间趋势
2. **getDepartmentYearlyRanking** - 年度排名
3. **getDepartmentMonthlyRanking** - 月度排名
4. **getMultiDepartmentComparison** - 多部门对比
5. **getHighFrequencyPeriods** - 高频时段

### 新增文件

- `src/pages/TrendAnalysisPage.tsx` - 分析页面（21KB）
- `src/db/api.ts` - 新增328行代码
- `src/routes.tsx` - 添加路由配置

### 依赖库

- **Recharts** - 图表库（已安装）
- **React** - 前端框架
- **TypeScript** - 类型系统

## ⚠️ 注意事项

1. **部门选择限制：** 最多5个部门
2. **数据范围：** 近6年数据
3. **性能考虑：** 大量数据可能影响加载速度
4. **去重规则：** 按部门+日期去重
