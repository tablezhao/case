## 📋 功能概述

**更新时间**：2025-12-09  
**功能模块**：网站基本信息配置  
**新增功能**：浏览器标签标题配置

---

## 🎯 功能详情

### 浏览器标签标题配置 ⭐

#### 功能描述

在网站基本信息配置模块中新增了"浏览器标签标题"字段，管理员可以自定义浏览器标签页显示的标题。

**默认标题**：合规通 Case Wiki

#### 功能特点

✅ **灵活配置**
- 可以自定义浏览器标签显示的标题
- 支持最多100个字符
- 建议不超过60个字符以确保完整显示

✅ **实时生效**
- 修改后保存即可生效
- 刷新页面后显示新标题
- 无需重启服务器

✅ **智能默认**
- 如果不填写，自动使用默认标题"合规通 Case Wiki"
- 确保浏览器标签始终有标题显示

✅ **友好提示**
- 实时显示字符数量
- 提供字符长度建议
- 显示默认标题提示

## 📍 使用方法

### 1. 进入配置页面

1. **登录后台管理系统**
   - 使用管理员账号登录

2. **进入网站基本信息配置**
   - 点击"网站基本信息配置"菜单
   - 或从管理后台首页进入

### 2. 修改浏览器标题

1. **找到浏览器标签标题字段**
   - 位于"网站备用名称/简称"字段下方
   - 标签为"浏览器标签标题"

2. **输入新标题**
   - 在输入框中输入想要显示的标题
   - 例如："合规通 Case Wiki"
   - 或："App监管案例查询平台"

3. **查看字符统计**
   - 输入框下方显示当前字符数
   - 格式：当前：XX/100
   - 建议不超过60个字符

4. **保存更改**
   - 点击"保存更改"按钮
   - 等待保存成功提示

### 3. 验证效果

1. **刷新页面**
   - 保存后刷新浏览器页面
   - 或打开新标签页访问网站

2. **查看浏览器标签**
   - 查看浏览器标签页的标题
   - 确认显示为新设置的标题

## 💡 使用建议

### 1. 标题长度建议

**推荐长度**：
- 最佳：10-30个字符
- 可接受：30-60个字符
- 不推荐：超过60个字符

**原因**：
- 浏览器标签宽度有限
- 标题过长会被截断
- 影响用户体验

**示例**：
```
✅ 推荐：
- "合规通 Case Wiki"（15字符）
- "App监管案例查询"（10字符）
- "合规通 - 案例查询平台"（13字符）

⚠️ 可接受：
- "合规通 - App监管案例查询平台"（18字符）
- "App违规监管案例查询系统"（15字符）

❌ 不推荐：
- "合规通 - 信息透明、查询便捷的App违规监管案例查询平台"（30+字符）
```

### 2. 标题内容建议

**包含要素**：
- 网站名称
- 核心功能
- 品牌标识

**示例**：
```
✅ 好的标题：
- "合规通 Case Wiki"
  - 包含品牌名称
  - 包含功能定位
  - 简洁明了

- "App监管案例查询"
  - 直接说明功能
  - 易于理解
  - 便于记忆

❌ 不好的标题：
- "网站"
  - 太过简单
  - 没有特色
  
- "这是一个用于查询App违规监管案例的专业平台系统"
  - 太长
  - 不简洁
```

### 3. SEO优化建议

**关键词**：
- 在标题中包含核心关键词
- 例如："App监管"、"案例查询"、"合规"

**格式**：
- 使用分隔符："-"、"|"、"·"
- 例如："合规通 - App监管案例"

**品牌**：
- 在标题中包含品牌名称
- 例如："合规通 Case Wiki"

## 🔍 常见问题

### Q1：修改后什么时候生效？

**A**：修改后立即生效，但需要刷新页面才能看到新标题。

**操作步骤**：
1. 在后台修改浏览器标题
2. 点击"保存更改"
3. 刷新浏览器页面
4. 查看浏览器标签的新标题

### Q2：如果不填写浏览器标题会怎样？

**A**：如果不填写，系统会自动使用默认标题"合规通 Case Wiki"。

**说明**：
- 默认标题已经过优化
- 适合大多数使用场景
- 如果没有特殊需求，可以不填写

### Q3：浏览器标题和网站主标题有什么区别？

**A**：两者用途不同：

| 项目 | 网站主标题 | 浏览器标签标题 |
|------|-----------|---------------|
| 显示位置 | 网站页面内 | 浏览器标签页 |
| 主要用途 | 页面展示 | 标签识别 |
| 长度建议 | 可以较长 | 建议简短 |
| 必填性 | 必填 | 可选 |

**示例**：
- 网站主标题："合规通"
- 浏览器标签标题："合规通 Case Wiki"

### Q4：标题可以包含特殊字符吗？

**A**：可以，但建议谨慎使用。

**推荐使用**：
- 中文字符：✅
- 英文字符：✅
- 数字：✅
- 常用符号："-"、"|"、"·"、"/"

**不推荐使用**：
- 表情符号：⚠️（可能显示异常）
- 特殊符号：❌（可能被过滤）
- HTML标签：❌（会被转义）

### Q5：可以为不同页面设置不同的标题吗？

**A**：目前不支持，所有页面使用统一的浏览器标题。

**说明**：
- 当前版本只支持全局统一标题
- 这是为了保持品牌一致性
- 如有特殊需求，可以联系技术支持

### Q6：修改标题会影响SEO吗？

**A**：会有一定影响，建议谨慎修改。

**影响**：
- 搜索引擎会重新索引新标题
- 可能影响搜索排名
- 需要一段时间才能生效

**建议**：
- 确定好标题后尽量不要频繁修改
- 包含核心关键词
- 保持标题的相关性

### Q7：标题显示不完整怎么办？

**A**：这是因为标题太长，被浏览器截断了。

**解决方法**：
1. 缩短标题长度
2. 使用更简洁的表达
3. 去掉不必要的修饰词
4. 建议控制在60个字符以内

**示例**：
```
❌ 太长：
"合规通 - 信息透明、查询便捷的App违规监管案例查询平台"

✅ 简化后：
"合规通 Case Wiki"
```

## 🎯 技术实现

### 数据库结构

**表名**：`site_settings`

**新增字段**：
```sql
browser_title text CHECK (char_length(browser_title) <= 100)
```

**字段说明**：
- 类型：文本
- 可空：是
- 最大长度：100个字符
- 默认值：NULL（使用默认标题）

### 前端实现

**Hook**：`useBrowserTitle`

**功能**：
- 从数据库读取浏览器标题配置
- 动态更新 `document.title`
- 如果没有配置，使用默认标题

**代码位置**：
- Hook：`src/hooks/useBrowserTitle.ts`
- 使用：`src/App.tsx`

**工作流程**：
```
1. 应用启动
   ↓
2. useBrowserTitle Hook 执行
   ↓
3. 从数据库读取配置
   ↓
4. 更新 document.title
   ↓
5. 浏览器标签显示新标题
```
