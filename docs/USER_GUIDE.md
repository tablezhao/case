# 数据展示优化 - 用户指南

## 📊 优化内容

### 1. 统计说明改为悬浮提示
**之前**：统计说明固定显示在页面上，占用空间  
**现在**：鼠标悬停在 ⓘ 图标上即可查看详细说明

### 2. "持平"显示更清晰
**之前**：`持平 较上月`  
**现在**：`持平 (0%) 较上月`

## 🎯 如何使用

### 查看统计口径说明

1. 找到数据卡片标题旁的 **ⓘ 图标**
2. 将鼠标移动到图标上
3. 自动显示详细说明
4. 移开鼠标，说明自动隐藏

### 理解数据关系

**通报频次**：
- 定义：通报活动的次数
- 计算：按"部门+日期"去重
- 示例：2025-12-04，某部门发布通报 = 1次

**涉及应用**：
- 定义：被通报的唯一应用数量
- 计算：按应用名称去重
- 示例：81条记录 → 69个应用（去重后）

**关系**：
- 1次通报可以涉及多个应用
- 同一应用在多个平台被通报只计算1次

---

## ❓ 常见问题

### Q1：为什么通报频次=1，但涉及应用=69？
**答**：1次通报活动可以涉及多个应用。  
例如：2025-12-04，国家计算机病毒应急处理中心发布1次通报，涉及69个应用。

### Q2：为什么显示"持平"？
**答**：因为去重后的应用数量没有变化。
- 12月：69个应用（81条记录）
- 11月：69个应用（71条记录）
- 结论：去重后应用数量保持不变 → 持平 (0%)

### Q3：什么是"去重统计"？
**答**：同一应用在多个平台被通报，只计算1次。  
例如：应用A在微信小程序和支付宝小程序都被通报 → 计为1个应用
