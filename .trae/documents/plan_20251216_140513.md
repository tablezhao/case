## Vercel Analytics 接入方案

### 项目现状
- 已安装 `@vercel/analytics` 依赖（版本 1.6.1）
- 未在代码中初始化或使用 Vercel Analytics

### 接入步骤

#### 1. 确认接入位置
项目是 React 应用，推荐将 `Analytics` 组件添加到应用根组件中。

#### 2. 选择接入方式
React 应用接入 Vercel Analytics 有两种方式：
- 使用 `<Analytics />` 组件（推荐）
- 使用 `inject()` 函数（适用于非 React 应用）

#### 3. 实施步骤

**步骤 1：在 `main.tsx` 中导入并使用 `Analytics` 组件**

修改 `src/main.tsx` 文件，添加 `Analytics` 组件到应用根节点。

**步骤 2：配置选项（可选）**
根据需要配置 `Analytics` 组件的选项，如 `mode`、`debug` 等。

**步骤 3：验证接入**
部署应用后，在 Vercel 控制台查看 Analytics 数据。

### 代码修改

```typescript
// src/main.tsx
import { StrictMode } from "react";
import { createRoot } from "react-dom/client";
import "./index.css";
import App from "./App.tsx";
import { AppWrapper } from "./components/common/PageMeta.tsx";
// 导入 Vercel Analytics
import { Analytics } from "@vercel/analytics/react";

createRoot(document.getElementById("root")!).render(
  <StrictMode>
    <AppWrapper>
      <App />
      {/* 添加 Analytics 组件 */}
      <Analytics />
    </AppWrapper>
  </StrictMode>
);
```

### 预期效果

- 应用启动后，Vercel Analytics 会自动初始化
- 收集页面访问、事件等数据
- 在 Vercel 控制台可以查看分析数据

### 注意事项

1. **部署环境**：Vercel Analytics 主要在部署到 Vercel 的环境中生效，本地开发环境可能不会发送数据
2. **隐私设置**：可以通过配置选项控制数据收集行为
3. **调试模式**：开发时可以开启 `debug` 选项查看日志

### 可选配置

如果需要自定义配置，可以添加以下选项：

```typescript
<Analytics 
  mode="production" // 默认为 "auto"，根据环境自动切换
  debug={false} // 是否开启调试模式
  // 其他配置选项
/>
```

通过以上步骤，即可完成 Vercel Analytics 的接入。
